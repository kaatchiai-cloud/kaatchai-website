# Claude Code Instructions for Stori

## **NEVER LIE. DO NOT MAKE FALSE CLAIMS ABOUT CODE WITHOUT CHECKING FIRST. IF YOU HAVEN'T READ THE CODE, SAY SO.**

## Critical Rules

1. **If you don't know, always check.** Never guess model names, API parameters, endpoints, or technical details. Verify by reading existing code, docs, or asking the user.

2. **Before proposing a solution, check that all user requirements are satisfied.** Re-read the user's instructions, list every requirement, and confirm your code addresses each one before writing it.

3. **Don't repeat mistakes.** If corrected, fix it right the first time. Don't iterate through the same error multiple times.

4. **Don't assume.** If unclear about what the user wants, ask. Don't fill in gaps with guesses.

5. **NEVER LIE.** When asked "does X do Y?" — READ THE CODE FIRST. If you haven't checked, say "I haven't checked yet." Never fabricate an answer.
